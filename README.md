# 🛒 Retail Customer Segmentation — Tech-orda

## 🎯 Цель проекта
Проанализировать поведение клиентов интернет-магазина, выявить ключевые сегменты покупателей и определить факторы, влияющие на их ценность для бизнеса.

## 🏢 Бизнес-задача
Ритейлер использует единую маркетинговую стратегию для всех клиентов — это приводит к потере высокоценных покупателей и неэффективным расходам на рекламу. Необходимо сегментировать аудиторию и выработать отдельный подход к каждой группе.

## ❓ Бизнес-вопросы
1. Какой сегмент клиентов приносит наибольший доход и чем он отличается от остальных?
2. Как регион и способ оплаты влияют на средний чек?
3. Снижает ли частое использование скидок долгосрочную ценность клиента?
4. Можно ли предсказать ежемесячные траты клиента по его характеристикам?
5. Есть ли значимая разница в поведении между городскими и сельскими покупателями?

## 📦 Данные

**Источник:** retail_customer_segmentation.csv  
**Объём:** 50 000 строк · 14 колонок

| Колонка | Тип | Описание |
|---|---|---|
| `customer_id` | int | Уникальный ID клиента |
| `age` | int | Возраст |
| `annual_income` | float | Годовой доход |
| `months_active` | int | Кол-во месяцев активности |
| `avg_monthly_spend` | float | Средние траты в месяц |
| `purchase_frequency` | float | Частота покупок |
| `avg_order_value` | float | Средний чек |
| `discount_usage_rate` | float | Доля покупок со скидкой |
| `return_rate` | float | Уровень возвратов |
| `browsing_time_minutes` | float | Время на сайте (мин) |
| `support_interactions` | float | Обращений в поддержку |
| `payment_method` | str | Способ оплаты (Card / Wallet / UPI) |
| `region` | str | Регион (Urban / Semi-Urban / Rural) |
| `customer_segment` | str | **Целевая переменная** (Occasional / Regular / Loyal / High_Value) |

**Пропуски:** ~5–8% в числовых колонках — будут заполнены медианой.

## 🗺️ План обработки данных

**Шаг 1 — Загрузка и первичный осмотр**
- Размер, типы данных, первые строки
- Количество уникальных значений по категориальным колонкам

**Шаг 2 — Очистка**
- Поиск и удаление дублей
- Заполнение пропусков (числовые → медиана, категориальные → мода)
- Приведение типов данных

**Шаг 3 — Разведочный анализ (EDA)**
- Распределение клиентов по сегментам
- Гистограммы: возраст, доход, средний чек
- Корреляционная матрица числовых признаков
- Анализ по регионам и способам оплаты

**Шаг 4 — Метрики**
- Средний LTV по сегменту
- Средний чек и частота покупок по регионам
- Уровень возвратов и скидок по сегментам

**Шаг 5 — Линейная регрессия**
- Цель: предсказать `avg_monthly_spend`
- Признаки: возраст, доход, частота, месяцы активности
- Метрики качества: R², MAE

**Шаг 6 — Проверка гипотез**
- Гипотеза 1: Urban-клиенты тратят больше Rural (t-test)
- Гипотеза 2: Высокий discount_usage снижает avg_order_value (t-test)

**Шаг 7 — Дашборд Power BI**
- KPI: выручка, средний чек, кол-во клиентов
- Разбивка по сегментам и регионам
- Фильтры по payment_method и region
